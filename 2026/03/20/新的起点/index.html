# Hexo博客常用命令归纳（速查版） 为了方便你日常管理博客，我把Hexo最核心、最常用的命令按「场景分类」整理好了，附说明和实用组合用法，新手也能直接套用👇 ## 一、基础环境&初始化（首次/迁移必用） | 命令 | 作用 | 适用场景 | |------|------|----------| | npm install hexo-cli -g | 全局安装Hexo命令行工具 | 新电脑首次配置、hexo命令报错时 | | npm install | 安装项目依赖 | 迁移博客文件后、新增插件/主题后 | | hexo init [文件夹名] | 初始化新的Hexo项目 | 从零创建新博客时（你已有项目，无需执行） | ## 二、本地开发&预览（日常写文章必用） ### 1. 核心命令 | 命令 | 简写 | 作用 | |------|------|------| | hexo clean | - | 清除缓存文件和生成的静态文件（解决样式/内容不更新问题） | | hexo generate | hexo g | 根据源文件生成静态博客页面（生成到public目录） | | hexo server | hexo s | 启动本地预览服务器（默认地址：http://localhost:4000） | ### 2. 实用组合命令（一键操作，推荐收藏） bash # 清理缓存 + 生成 + 启动本地服务（日常写文章最常用） hexo clean && hexo g && hexo s # 启动服务时自定义端口（默认4000被占用时） hexo s -p 5000 # 访问地址变为 http://localhost:5000 # 启动服务并实时监听文件变化（修改后自动刷新） hexo s -w ## 三、文章/页面管理（创作内容必用） ### 1. 创建内容 | 命令 | 作用 | 说明 | |------|------|------| | hexo new "文章标题" | 创建新博客文章 | 生成到source/_posts/文章标题.md | | hexo new page "页面名称" | 创建独立页面（如关于页、归档页） | 生成到source/页面名称/index.md | | hexo new draft "草稿标题" | 创建草稿文章 | 生成到source/_drafts/，默认不生成到静态文件 | ### 2. 草稿发布 bash # 将草稿转为正式文章（发布时用） hexo publish "草稿标题" ## 四、部署上线（发布到GitHub/Gitee等） ### 1. 基础部署命令 bash # 清理缓存 + 生成 + 部署（发布上线标准流程） hexo clean && hexo g && hexo d # 仅执行部署（已生成静态文件时） hexo d # 等价于 hexo deploy ### 2. 部署相关配置 - 部署前需确保： 1. 安装部署插件：npm install hexo-deployer-git --save 2. _config.yml中配置好仓库信息（示例）： yaml deploy: type: git repo: git@github.com:你的用户名/你的仓库名.git # SSH地址（推荐） branch: gh-pages # GitHub Pages默认分支 ## 五、进阶/应急命令 | 命令 | 作用 | 适用场景 | |------|------|----------| | hexo version | 查看Hexo及依赖版本 | 排查版本兼容问题 | | npx hexo xxx | 调用本地Hexo执行命令 | 不想全局安装Hexo时（替代hexo xxx） | | hexo g -d | 生成文件并直接部署 | 简化部署步骤（等价于hexo g && hexo d） | | hexo s -s | 静态模式启动服务 | 模拟线上环境预览（加载public目录文件） | ## 六、命令使用小贴士 1. 执行位置：所有命令都要在「博客根目录」下执行（包含package.json、_config.yml的目录）； 2. 缓存问题：修改主题配置、文章内容后，先执行hexo clean再预览，避免旧缓存干扰； 3. 快捷键：本地服务启动后，按Ctrl+C可停止服务； 4. 中文标题：创建文章时标题含中文，建议用引号包裹（如hexo new "我的第一篇Hexo博客"）。 --- ### 总结 1. 日常开发：核心组合 hexo clean && hexo g && hexo s，搞定本地预览； 2. 发布上线：核心组合 hexo clean && hexo g && hexo d，一键部署到线上； 3. 内容管理：创建文章用 hexo new "标题"，创建草稿用 hexo new draft "标题"，发布草稿用 hexo publish "标题"。 可以把这份命令清单保存到博客目录下的README.md，下次用的时候直接查看，不用再反复查资料～
